# IoT Monitor 机器人监测系统 - 启动文档

## 项目概述

本项目是一个基于 **Django 6.0 + Vue 3 + Bokeh** 的物联网机器人监测与数据管理系统，包含：

- **后端服务**：Django REST Framework + Celery + Bokeh Server
- **前端界面**：Vue 3 + Vite + Element Plus
- **可视化**：Bokeh 交互式图表 + ECharts
- **异步任务**：Celery + Redis
- **实时通信**：WebSocket (Django Channels)

## 启动前准备

### 1. 安装依赖

#### 后端依赖安装

```bash
# 激活虚拟环境（Windows）
cd C:\robotUI
venv57\Scripts\activate

# 激活虚拟环境（Linux/Mac）
cd /path/to/sg5.7/backend
source venv/bin/activate

# 安装 Python 依赖
pip install -r requirements.txt
```

#### 前端依赖安装

```bash
# 前端1（5173端口）
cd C:\robotUI\sg\sg\frontend
npm install

# 前端2（5174端口）
cd C:\robotUI\frontend-vue2
npm install
```

### 2. 数据库初始化

```bash
# 激活虚拟环境后
cd C:\robotUI\backend57

# 执行数据库迁移
python manage.py migrate

# 创建超级用户（可选）
python manage.py createsuperuser

# 初始化机器人平台数据
python manage.py seed_robot_platform
```

### 3. 配置环境变量

创建 `.env` 文件（参考 `.env.example`）：

```env
# Django 配置
SECRET_KEY=your-secret-key-here
DEBUG=True
ALLOWED_HOSTS=*

# 数据库配置
DB_NAME=iot_monitor
DB_USER=root
DB_PASSWORD=your-password
DB_HOST=localhost
DB_PORT=3306

# Redis 配置
REDIS_HOST=127.0.0.1
REDIS_PORT=6379

# Celery 配置
CELERY_BROKER_URL=redis://127.0.0.1:6379/0
CELERY_RESULT_BACKEND=redis://127.0.0.1:6379/0

# Bokeh Server 配置
BI_BOKEH_USE_SERVER=1
BI_BOKEH_SERVER_PORT=5008
```

### 4. 启动必要服务

#### 启动 MySQL

```bash
# Windows
net start MySQL

# Linux/Mac
sudo systemctl start mysql
# 或
sudo service mysql start
```

#### 启动 Redis

```bash
# Windows
redis-server

# Linux/Mac
redis-server
# 或
sudo systemctl start redis
```

## 常见问题排查

### 1. 端口被占用

**问题**：`Error: That port is already in use`

**解决方案**：

```bash
# Windows 查找占用端口的进程
netstat -ano | findstr :8001
taskkill /PID <进程ID> /F

# Linux/Mac 查找占用端口的进程
lsof -i :8001
kill -9 <进程ID>
```

### 2. MySQL 连接失败

**问题**：`Can't connect to MySQL server`

**解决方案**：

```bash
# 检查 MySQL 服务是否启动
# Windows
net start | findstr MySQL

# Linux/Mac
sudo systemctl status mysql

# 检查数据库配置
# 确保 .env 文件中的 DB_USER、DB_PASSWORD、DB_HOST、DB_PORT 正确
```

### 3. Redis 连接失败

**问题**：`Error connecting to Redis`

**解决方案**：

```bash
# 检查 Redis 服务
redis-cli ping
# 应返回: PONG

# 启动 Redis（如果未启动）
redis-server
```

### 4. Celery Worker 无响应

**问题**：任务无法执行

**解决方案**：

```bash
# 检查 Celery 日志
# 使用 -l debug 参数获取详细日志
python -m celery -A iot_monitor worker -l debug --pool=solo

# 清除旧的 Celery 状态
celery -A iot_monitor purge
```

### 5. 前端无法连接后端

**问题**：`Network Error` / `CORS Error`

**解决方案**：

```bash
# 检查后端是否正常运行
curl http://localhost:8001/api/devices/

# 检查 CORS 配置
# 确认 settings.py 中 CORS_ALLOW_ALL_ORIGINS = True
```

### 6. Bokeh Server 无法访问

**问题**：BI 页面加载失败

**解决方案**：

```bash
# 检查 Bokeh Server 是否启动
curl http://localhost:5008

# 检查配置
# 确保 BI_BOKEH_USE_SERVER=1
# 确保 BI_BOKEH_SERVER_PORT=5008
```

## 开发模式快捷启动脚本

### Windows 批处理脚本

创建 `start_all.bat`：

```batch
@echo off
echo Starting IoT Monitor System...

start "Django Backend" cmd /k "cd C:\robotUI && venv57\Scripts\activate && cd backend57 && python manage.py runserver 8001"
timeout /t 3 /nobreak

start "Bokeh Server" cmd /k "cd C:\robotUI && venv57\Scripts\activate && cd backend57 && python manage.py run_bi_bokeh_server --port 5008"
timeout /t 3 /nobreak

start "Frontend 5173" cmd /k "cd C:\robotUI\sg\sg\frontend && npm run dev"
timeout /t 3 /nobreak

start "Frontend 5174" cmd /k "cd C:\robotUI\frontend-vue2 && npm run dev"
timeout /t 3 /nobreak

start "Celery Worker" cmd /k "cd C:\robotUI && venv57\Scripts\activate && cd backend57 && python -m celery -A iot_monitor worker -l info --pool=solo"
timeout /t 2 /nobreak

start "Celery Beat" cmd /k "cd C:\robotUI && venv57\Scripts\activate && cd backend57 && python -m celery -A iot_monitor beat -l info --schedule=C:\robotUI\celerybeat-schedule --pidfile=C:\robotUI\celerybeat.pid"

echo All services started!
```

### Linux/Mac Shell 脚本

创建 `start_all.sh`：

```bash
#!/bin/bash

echo "Starting IoT Monitor System..."

# Django Backend
gnome-terminal -- bash -c "cd /path/to/sg5.7/backend && source venv/bin/activate && python manage.py runserver 8001; exec bash"
sleep 3

# Bokeh Server
gnome-terminal -- bash -c "cd /path/to/sg5.7/backend && source venv/bin/activate && python manage.py run_bi_bokeh_server --port 5008; exec bash"
sleep 3

# Frontend 5173
gnome-terminal -- bash -c "cd /path/to/sg5.7/frontend && npm run dev; exec bash"
sleep 3

# Frontend 5174
gnome-terminal -- bash -c "cd /path/to/sg5.7/frontend-vue && npm run dev; exec bash"
sleep 3

# Celery Worker
gnome-terminal -- bash -c "cd /path/to/sg5.7/backend && source venv/bin/activate && celery -A iot_monitor worker -l info --pool=solo; exec bash"
sleep 2

# Celery Beat
gnome-terminal -- bash -c "cd /path/to/sg5.7/backend && source venv/bin/activate && celery -A iot_monitor beat -l info --scheduler django_celery_beat.schedulers:DatabaseScheduler; exec bash"

echo "All services started!"
```

## 生产环境部署

### 使用 Docker Compose

```yaml
# docker-compose.yml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: your-password
      MYSQL_DATABASE: iot_monitor
    volumes:
      - mysql_data:/var/lib/mysql
    ports:
      - "3306:3306"

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  backend:
    build: ./backend
    command: gunicorn iot_monitor.wsgi:application --bind 0.0.0.0:8001
    volumes:
      - ./backend:/app
    ports:
      - "8001:8001"
    depends_on:
      - mysql
      - redis

  celery-worker:
    build: ./backend
    command: celery -A iot_monitor worker -l info
    volumes:
      - ./backend:/app
    depends_on:
      - redis

  celery-beat:
    build: ./backend
    command: celery -A iot_monitor beat -l info --scheduler django_celery_beat.schedulers:DatabaseScheduler
    volumes:
      - ./backend:/app
    depends_on:
      - redis

  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend

volumes:
  mysql_data:
```

启动命令：

```bash
docker-compose up -d
```
