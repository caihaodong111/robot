## 通用说明

### 请求格式

```
GET /api/{module}/{action}/?param1=value1&param2=value2
POST /api/{module}/{action}/
Content-Type: application/json

{
  "field1": "value1",
  "field2": "value2"
}
```

### 响应格式

**成功响应**：
```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

**失败响应**：
```json
{
  "code": 400,
  "message": "error message",
  "data": null
}
```

### 分页参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `page` | integer | 否 | 页码，默认 1 |
| `page_size` | integer | 否 | 每页数量，默认 20 |

### 分页响应

```json
{
  "count": 100,
  "next": "http://localhost:8001/api/devices/?page=2",
  "previous": null,
  "results": [ ... ]
}
```

## 认证接口

### 基础路径

```
/api/auth/
```

### 1. 用户登录

**接口**：`POST /api/auth/login/`

**请求参数**：
```json
{
  "username": "admin",
  "password": "password"
}
```

**响应示例**：
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "user_id": 1,
    "username": "admin",
    "token": "xxxxxxxxxxxx"
  }
}
```

### 2. 用户登出

**接口**：`POST /api/auth/logout/`

**请求头**：
```
Authorization: Token xxxxxxxxxxxx
```

**响应示例**：
```json
{
  "code": 200,
  "message": "登出成功"
}
```

### 3. 获取用户信息

**接口**：`GET /api/auth/profile/`

**请求头**：
```
Authorization: Token xxxxxxxxxxxx
```

**响应示例**：
```json
{
  "code": 200,
  "data": {
    "user_id": 1,
    "username": "admin",
    "email": "admin@example.com",
    "is_active": true
  }
}
```

## 设备管理接口

### 基础路径

```
/api/devices/
```

### 1. 获取设备列表

**接口**：`GET /api/devices/devices/`

**查询参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `page` | integer | 否 | 页码 |
| `page_size` | integer | 否 | 每页数量 |
| `status` | string | 否 | 设备状态筛选 |
| `search` | string | 否 | 搜索关键词 |

**响应示例**：
```json
{
  "count": 10,
  "next": null,
  "previous": null,
  "results": [
    {
      "id": 1,
      "name": "Device-001",
      "type": "sensor",
      "status": "online",
      "location": "Workshop A",
      "created_at": "2024-01-01T00:00:00Z",
      "updated_at": "2024-01-01T00:00:00Z"
    }
  ]
}
```

### 2. 获取设备详情

**接口**：`GET /api/devices/devices/{id}/`

**路径参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | integer | 是 | 设备 ID |

**响应示例**：
```json
{
  "id": 1,
  "name": "Device-001",
  "type": "sensor",
  "status": "online",
  "location": "Workshop A",
  "description": "Temperature sensor",
  "metadata": {},
  "created_at": "2024-01-01T00:00:00Z",
  "updated_at": "2024-01-01T00:00:00Z"
}
```

### 3. 创建设备

**接口**：`POST /api/devices/devices/`

**请求参数**：
```json
{
  "name": "Device-002",
  "type": "sensor",
  "location": "Workshop B",
  "description": "New device"
}
```

**响应示例**：
```json
{
  "id": 2,
  "name": "Device-002",
  "type": "sensor",
  "status": "offline",
  "location": "Workshop B",
  "created_at": "2024-01-02T00:00:00Z"
}
```

### 4. 更新设备

**接口**：`PUT /api/devices/devices/{id}/`

**请求参数**：
```json
{
  "name": "Device-002-Updated",
  "status": "online"
}
```

### 5. 删除设备

**接口**：`DELETE /api/devices/devices/{id}/`

### 6. 切换设备状态

**接口**：`POST /api/devices/devices/{id}/toggle-status/`

**响应示例**：
```json
{
  "code": 200,
  "message": "状态已切换",
  "data": {
    "id": 1,
    "status": "online"
  }
}
```

## 监测数据接口

### 基础路径

```
/api/monitoring/
```

### 1. 获取传感器数据列表

**接口**：`GET /api/monitoring/data/`

**查询参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `device_id` | integer | 否 | 设备 ID 筛选 |
| `start_time` | string | 否 | 开始时间 (ISO 8601) |
| `end_time` | string | 否 | 结束时间 (ISO 8601) |
| `page` | integer | 否 | 页码 |
| `page_size` | integer | 否 | 每页数量 |

**响应示例**：
```json
{
  "count": 100,
  "results": [
    {
      "id": 1,
      "device_id": 1,
      "sensor_type": "temperature",
      "value": 25.5,
      "unit": "°C",
      "timestamp": "2024-01-01T00:00:00Z"
    }
  ]
}
```

### 2. 上传监测数据

**接口**：`POST /api/monitoring/upload/`

**请求参数**：
```json
{
  "device_id": 1,
  "sensor_type": "temperature",
  "value": 25.5,
  "unit": "°C",
  "timestamp": "2024-01-01T00:00:00Z"
}
```

### 3. 查询监测数据

**接口**：`POST /api/monitoring/query/`

**请求参数**：
```json
{
  "device_ids": [1, 2, 3],
  "sensor_types": ["temperature", "humidity"],
  "start_time": "2024-01-01T00:00:00Z",
  "end_time": "2024-01-02T00:00:00Z",
  "aggregation": "avg",
  "interval": "1h"
}
```

**聚合类型**：
- `avg` - 平均值
- `min` - 最小值
- `max` - 最大值
- `sum` - 求和
- `count` - 计数

### 4. 导出监测数据

**接口**：`POST /api/monitoring/export/`

**请求参数**：
```json
{
  "device_ids": [1, 2],
  "start_time": "2024-01-01T00:00:00Z",
  "end_time": "2024-01-02T00:00:00Z",
  "format": "csv"
}
```

**导出格式**：
- `csv` - CSV 格式
- `excel` - Excel 格式
- `json` - JSON 格式

### 5. 获取实时数据

**接口**：`GET /api/monitoring/realtime/{device_id}/`

**响应示例**：
```json
{
  "device_id": 1,
  "data": [
    {
      "sensor_type": "temperature",
      "value": 25.5,
      "unit": "°C",
      "timestamp": "2024-01-01T00:00:00Z"
    }
  ]
}
```

### 6. 获取统计数据

**接口**：`GET /api/monitoring/statistics/{device_id}/`

**查询参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `start_time` | string | 否 | 开始时间 |
| `end_time` | string | 否 | 结束时间 |

**响应示例**：
```json
{
  "device_id": 1,
  "statistics": {
    "temperature": {
      "avg": 25.3,
      "min": 20.1,
      "max": 30.5,
      "count": 1000
    }
  }
}
```

## 报警管理接口

### 基础路径

```
/api/alerts/
```

### 1. 获取报警规则列表

**接口**：`GET /api/alerts/rules/`

**响应示例**：
```json
{
  "count": 5,
  "results": [
    {
      "id": 1,
      "name": "高温报警",
      "device_id": 1,
      "condition": "temperature > 30",
      "severity": "warning",
      "enabled": true,
      "created_at": "2024-01-01T00:00:00Z"
    }
  ]
}
```

### 2. 创建报警规则

**接口**：`POST /api/alerts/rules/`

**请求参数**：
```json
{
  "name": "高温报警",
  "device_id": 1,
  "condition": "temperature > 30",
  "severity": "warning",
  "notification_method": "email"
}
```

**严重级别**：
- `info` - 信息
- `warning` - 警告
- `critical` - 严重

### 3. 获取报警记录列表

**接口**：`GET /api/alerts/records/`

**查询参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `rule_id` | integer | 否 | 规则 ID 筛选 |
| `status` | string | 否 | 状态筛选 |
| `severity` | string | 否 | 严重级别筛选 |
| `start_time` | string | 否 | 开始时间 |
| `end_time` | string | 否 | 结束时间 |

**状态类型**：
- `pending` - 待处理
- `acknowledged` - 已确认
- `resolved` - 已解决

### 4. 确认报警

**接口**：`POST /api/alerts/records/{id}/acknowledge/`

**请求参数**：
```json
{
  "note": "正在处理中"
}
```

### 5. 解决报警

**接口**：`POST /api/alerts/records/{id}/resolve/`

**请求参数**：
```json
{
  "note": "问题已解决"
}
```

### 6. 获取报警统计

**接口**：`GET /api/alerts/statistics/`

**查询参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `start_time` | string | 否 | 开始时间 |
| `end_time` | string | 否 | 结束时间 |

**响应示例**：
```json
{
  "total": 100,
  "by_status": {
    "pending": 10,
    "acknowledged": 20,
    "resolved": 70
  },
  "by_severity": {
    "info": 50,
    "warning": 30,
    "critical": 20
  }
}
```

## WebSocket 接口

### 连接地址

```
ws://localhost:8001/ws/monitoring/
```

### 消息格式

**客户端发送**：
```json
{
  "action": "subscribe",
  "device_id": 1
}
```

**服务端推送**：
```json
{
  "type": "sensor_data",
  "device_id": 1,
  "data": {
    "sensor_type": "temperature",
    "value": 25.5,
    "timestamp": "2024-01-01T00:00:00Z"
  }
}
```

### 支持的消息类型

| 类型 | 说明 |
|------|------|
| `sensor_data` | 传感器数据推送 |
| `alert` | 报警通知 |
| `device_status` | 设备状态变更 |
| `system` | 系统通知 |

## 错误码说明

### HTTP 状态码

| 状态码 | 说明 |
|--------|------|
| 200 | 请求成功 |
| 201 | 创建成功 |
| 204 | 删除成功 |
| 400 | 请求参数错误 |
| 401 | 未认证 |
| 403 | 权限不足 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 业务错误码

| 错误码 | 说明 |
|--------|------|
| 1001 | 用户名或密码错误 |
| 1002 | Token 已过期 |
| 1003 | 权限不足 |
| 2001 | 设备不存在 |
| 2002 | 设备状态异常 |
| 3001 | 报警规则不存在 |
| 3002 | 报警记录不存在 |
| 4001 | 机器人不存在 |

## 接口调用示例

### Python 示例

```python
import requests

# 基础配置
BASE_URL = "http://localhost:8001/api"

# 获取设备列表
response = requests.get(f"{BASE_URL}/devices/devices/")
print(response.json())

# 创建设备
data = {
    "name": "Device-003",
    "type": "sensor",
    "location": "Workshop C"
}
response = requests.post(f"{BASE_URL}/devices/devices/", json=data)
print(response.json())
```

### JavaScript 示例

```javascript
// 基础配置
const BASE_URL = 'http://localhost:8001/api';

// 获取设备列表
fetch(`${BASE_URL}/devices/devices/`)
  .then(response => response.json())
  .then(data => console.log(data));

// 创建设备
const data = {
  name: 'Device-003',
  type: 'sensor',
  location: 'Workshop C'
};
fetch(`${BASE_URL}/devices/devices/`, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(data)
})
  .then(response => response.json())
  .then(data => console.log(data));
```

### cURL 示例

```bash
# 获取设备列表
curl -X GET http://localhost:8001/api/devices/devices/

# 创建设备
curl -X POST http://localhost:8001/api/devices/devices/ \
  -H "Content-Type: application/json" \
  -d '{"name": "Device-003", "type": "sensor", "location": "Workshop C"}'

# 用户登录
curl -X POST http://localhost:8001/api/auth/login/ \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "password"}'
```
