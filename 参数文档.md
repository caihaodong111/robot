# 参数文档（sg5.7）

## 1. 通用请求参数

### 1.1 鉴权 Header

- `Authorization: Token <token>`

> 注：后端 `backend/iot_monitor/settings.py` 当前未配置 `DEFAULT_AUTHENTICATION_CLASSES`，如需严格启用 Token 鉴权，建议补充 `TokenAuthentication`（见 `开发文档.md`）。

### 1.2 分页参数

- `page`：int，从 1 开始
- `page_size`：int（`robots` 的 `StandardResultsSetPagination` 最大 100）

### 1.3 时间参数格式

- `DateTime（ISO8601）`：例如 `2026-03-16T10:00:00+08:00`
- `Date（YYYY-MM-DD）`：例如 `2026-03-16`
- `export 专用（YYYY-MM-DD HH:MM:SS）`：例如 `2026-03-16 10:00:00`

## 2. 枚举与取值

### 2.1 Device

- `device_type`（`backend/devices/models.py`）
  - `temperature | humidity | light | pm25 | co2 | composite`
- `status`
  - `online | offline | maintenance | error`

### 2.2 AlertRule

- `sensor_type`（模型提供更全的枚举；任务检查目前仅使用部分类型）
  - 常用：`temperature | humidity | light_intensity | pm25 | co2`
- `condition`
  - `greater_than | less_than | equal | not_equal | between | outside`
- `severity`
  - `info | low | medium | high | critical`

### 2.3 RiskEvent

- `severity`：`low | medium | high | critical`
- `status`：`pending | acknowledged | resolved`

### 2.4 RobotComponent / RobotHighRiskSnapshot

- `level`：`H | M | L | T | C`
- `axisKey/axisKeys`：`A1..A7`
- `axisOk`：可传 `1/0/true/false/ok/high/...`（最终由 `parse_axis_ok` 解析）

## 3. 数据结构

### 3.1 登录请求 / 响应

**LoginRequest**

- `username` string 必填
- `password` string 必填

**LoginResponse**

- `message` string
- `user_id` int
- `username` string
- `token` string

### 3.2 Device

**DeviceCreate（POST /api/devices/devices/）**

- `name` string 必填
- `device_id` string 必填（全局唯一）
- `device_type` string 必填（见枚举）
- `location` string 必填
- `description` string 可选
- `config` object 可选（见 `DeviceConfig`）

**DeviceList（GET /api/devices/devices/ 的 results 元素）**

- `id` int
- `name` string
- `device_id` string
- `device_type` string
- `location` string
- `status` string
- `owner_name` string
- `last_active` datetime|null
- `sensor_count` int

**DeviceDetail（GET /api/devices/devices/{id}/）**

- `id` int
- `name` string
- `device_id` string
- `device_type` string
- `location` string
- `status` string
- `description` string
- `owner` int
- `owner_name` string
- `created_at` datetime
- `updated_at` datetime
- `last_active` datetime|null
- `config` object（见 `DeviceConfig`）

### 3.3 DeviceConfig

**DeviceConfig（读写）**

- `id` int
- `device` int
- `collect_interval` int（秒）
- `enable_alert` boolean
- `data_retention_days` int
- `auto_cleanup` boolean
- `extra_config` object
- `created_at` datetime
- `updated_at` datetime

### 3.4 SensorData

**SensorDataCreate（POST /api/monitoring/upload/）**

- `device` int 必填（设备表主键）
- `temperature` number|null
- `humidity` number|null
- `light_intensity` number|null
- `pm25` number|null
- `co2` number|null

**SensorData（输出）**

- `id` int
- `device` int
- `device_name` string
- `device_id` string（设备业务 id）
- `temperature` number|null
- `humidity` number|null
- `light_intensity` number|null
- `pm25` number|null
- `co2` number|null
- `timestamp` datetime
- `is_valid` boolean
- `error_message` string

### 3.5 AlertRule

**AlertRule（读写；序列化器为 `fields="__all__"`）**

- `id` int
- `name` string
- `device` int
- `device_name` string（只读）
- `sensor_type` string
- `condition` string
- `threshold_min` number|null
- `threshold_max` number|null
- `threshold` number|null
- `severity` string
- `enabled` boolean
- `description` string
- `cooldown_minutes` int
- `repeat_alert` boolean
- `repeat_interval_minutes` int
- `created_by` int（只读）
- `created_by_name` string（只读）
- `created_at` datetime（只读）
- `updated_at` datetime（只读）

> 注：`alerts/tasks.py` 当前只对 `threshold` 做比较，`between/outside/threshold_min/threshold_max` 暂未使用。

### 3.6 AlertRecord

**AlertRecord（只读；序列化器为 `fields="__all__"`）**

- `id` int
- `rule` int
- `rule_name` string（只读）
- `device` int
- `device_name` string（只读）
- `status` string
- `message` string
- `current_value` number
- `severity` string
- `triggered_at` datetime（只读）
- `acknowledged_at` datetime|null（只读）
- `acknowledged_by` int|null
- `acknowledged_by_name` string|null（只读）
- `resolved_at` datetime|null（只读）
- `resolved_by` int|null
- `resolved_by_name` string|null（只读）
- `notes` string
- `recovery_value` number|null
- `recovery_time` duration|null
- `notification_sent` boolean
- `notification_count` int

### 3.7 RobotGroup

- `id` int
- `key` string（车间/组 key）
- `name` string
- `expected_total` int
- `stats` object
  - `total` int
  - `highRisk` int
  - `disconnected` int

### 3.8 RobotComponent（列表）

（`GET /api/robots/components/?tab!=all` 使用列表序列化器）

- `id` int
- `group` string（group.key）
- `robot` string
- `shop` string|null
- `referenceNo` string|null
- `number` number|null
- `type` string|null
- `tech` string|null
- `mark` int
- `remark` string
- `a1..a7` string|null（轴状态字符串，常见值如 `high`）
- `p_change` number|null
- `level` string|null
- `isHighRisk` boolean
- `updated_at` datetime

### 3.9 RobotComponent（详情/编辑）

（字段量更大，严格对应 weeklyresult.csv 导入字段；输出包含 `referenceNo`）

**编辑常用可写字段（PATCH）**

- `reference` string（写入用；输出为 `referenceNo`）
- `number` number|null
- `mark` int
- `remark` string
- `level` string（空字符串会被忽略）

**主要输出字段（按组）**

- 基本字段：`robot/shop/type/tech/mark/remark/created_at/updated_at`
- 误差与温度：`error1_c1/tem1_m..tem7_m`
- 关节错误率：`a1_e_rate..a7_e_rate`
- RMS：`a1_rms..a7_rms`
- E 值：`a1_e..a7_e`
- Q 值：`q1..q7`
- 电流极值：`curr_a1_max..curr_a7_max`、`curr_a1_min..curr_a7_min`
- 轴状态字符串：`a1..a7`
- 其他：`p_change`、`level`、`isHighRisk`
