# 开发文档（sg5.7）

## 1. 项目概览

- 前端：`frontend/`（Vue3 + Vite + Element Plus）
- 后端：`backend/`（Django + DRF + Channels + Celery）
- 统一 API 前缀：`/api`（见 `backend/iot_monitor/urls.py`）

## 2. 本地开发环境要求

- Node.js：建议 `>= 18`
- Python：建议 `>= 3.10`
- MySQL：建议 `>= 8.0`
- Redis：建议 `>= 6.0`（Celery broker + Channels）

## 3. 后端启动（Django）

### 3.1 安装依赖

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 3.2 环境变量（.env）

后端会 `load_dotenv()`（见 `backend/iot_monitor/settings.py`），你可以在 `backend/.env` 配置：

**数据库**

- `DB_NAME`
- `DB_USER`
- `DB_PASSWORD`
- `DB_HOST`
- `DB_PORT`

**Redis / Celery**

- `REDIS_HOST`、`REDIS_PORT`
- `CELERY_BROKER_URL`、`CELERY_RESULT_BACKEND`

**机器人模块（可选）**

- 路径与 CSV：
  - `PATH_CONFIG_FILE`（默认 `backend/path_config.json`）
  - `WEEKLY_RESULT_FOLDER`（默认仓库根目录的上级）
  - `ROBOT_CONFIG_CSV`（默认 `机器人配置文件.csv`）
  - `REFERENCE_DICT_CSV`（默认 `dic information .csv`）
  - `ERROR_RATE_CSV_PATH`（错误率 CSV 目录）
  - `ERROR_RATE_CHART_PATH`（图表输出目录）
- MySQL 导入优化（可选）：
  - `MYSQL_LOCAL_INFILE=1`（允许 `LOAD DATA LOCAL INFILE`）
  - `ENABLE_MYSQL_LOAD_DATA=1`（允许周结果导入走 MySQL load 路径）
- 外部数据库（可选）：
  - SG（用于 `bi_robots` 回退）：`SG_DB_NAME/USER/PASSWORD/HOST/PORT`
  - Keypath 告警：`KEY_DB_NAME/USER/PASSWORD/HOST/PORT`

### 3.3 初始化数据库

```bash
cd backend
python manage.py makemigrations
python manage.py migrate
python manage.py createsuperuser  # 可选
```

### 3.4 启动服务

前端 `vite.config.js` 代理目标默认为 `http://127.0.0.1:8001`，建议二选一：

- 方案 A：后端跑在 8001

```bash
cd backend
python manage.py runserver 0.0.0.0:8001
```

- 方案 B：修改 `frontend/vite.config.js` 将代理改为 8000，然后：

```bash
cd backend
python manage.py runserver 0.0.0.0:8000
```

## 4. 异步任务（Celery）

数据上报会触发报警检查任务（见 `backend/alerts/tasks.py`），机器人数据也有定时同步（见 `backend/robots/tasks.py`）。

```bash
cd backend
celery -A iot_monitor worker -l info
celery -A iot_monitor beat -l info  # 可选：启用定时任务
```

依赖：Redis 可用（`CELERY_BROKER_URL`）。

## 5. WebSocket（Channels）

- 路由：`ws/realtime/<device_id>/`（见 `backend/monitoring/routing.py`）
- ASGI：`backend/iot_monitor/asgi.py` 使用懒加载 `URLRouter`

> 注意：当前仓库只实现了 consumer，尚未实现向组广播 `sensor_update` 的生产者逻辑（搜索不到 `group_send`）。

## 6. 前端启动（Vue3）

```bash
cd frontend
npm install
npm run dev
```

默认访问：`http://localhost:5173`（API 走 `/api` 代理到后端）。

## 7. 接口鉴权说明（Token）

前端会在请求头附带 `Authorization: Token <token>`（见 `frontend/src/utils/request.js`），但后端 `REST_FRAMEWORK['DEFAULT_AUTHENTICATION_CLASSES']` 当前为空（见 `backend/iot_monitor/settings.py`）。

如果你希望后端真正启用 Token 鉴权（并让 `request.user.is_authenticated` 生效），通常需要：

1) 在 `backend/iot_monitor/settings.py` 配置：

- `rest_framework.authentication.TokenAuthentication`
- （可选）`rest_framework.authentication.SessionAuthentication`

2) 执行迁移以创建 token 表：

```bash
cd backend
python manage.py migrate authtoken
```

3) 将需要登录的接口加上权限（例如 `IsAuthenticated`），或配置全局默认权限。

（以上为建议项，不会自动修改现有代码行为。）

## 8. 部署（Docker）

仓库根目录存在 `docker-compose.yml`（镜像：`sg-backend:v0.3`、`sg-frontend:v0.3`）：

- 后端对外：`8000`
- 前端对外：`8080`

如使用本地构建/自定义端口，请同步调整前端代理与运行端口。
