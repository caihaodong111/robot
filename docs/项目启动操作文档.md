# 项目启动操作文档（sg5.7）

本文档面向“从零把项目跑起来”的场景，覆盖本地开发启动方式。

## 1. 项目结构与服务说明

- 后端：`backend/`（Django + DRF + Channels + Celery）
  - HTTP/API：默认 `http://127.0.0.1:8001/`（可按需改为 8000）
  - WebSocket：`ws://<host>/ws/...`（由 Channels 提供，依赖 Redis）
- 前端：`frontend/`（Vue3 + Vite + Element Plus）
  - 开发模式：默认 `http://localhost:5173/`

关键文件：

- 入口说明：`README.md`、`开发文档.md`
- 后端启动入口：`backend/manage.py`
- 前端启动入口：`frontend/package.json`
- 本地代理：`frontend/vite.config.js`（默认把 `/api` 代理到 `http://127.0.0.1:8001`）

## 2. 环境要求

- Node.js：建议 `>= 18`
- Python：建议 `>= 3.10`
- MySQL：建议 `>= 8.0`（后端默认使用 MySQL）
- Redis：建议 `>= 6.0`（Channels/缓存/Celery broker）

## 3. 本地开发启动（推荐）

### 3.1 后端（Django）

1) 创建虚拟环境并安装依赖：

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

2) 配置环境变量：

- 后端会在 `backend/iot_monitor/settings.py` 中调用 `load_dotenv()`，默认读取 `backend/.env`。
- 如需自定义，请编辑/新建 `backend/.env`（示例字段见“3.3 环境变量模板”）。

3) 初始化数据库：

```bash
cd backend
source .venv/bin/activate
python manage.py makemigrations
python manage.py migrate
python manage.py createsuperuser  # 可选
```

4) 启动后端服务（与前端默认代理保持一致，推荐 8001）：

```bash
cd backend
source .venv/bin/activate
python manage.py runserver 0.0.0.0:8001
```

可选：启动异步任务（需要 Redis 可用）：

```bash
cd backend
source .venv/bin/activate
celery -A iot_monitor worker -l info
celery -A iot_monitor beat -l info  # 可选：启用定时任务
```

### 3.2 前端（Vue3）

```bash
cd frontend
npm install
npm run dev
```

访问：

- 前端：`http://localhost:5173/`
- 后端 API：`http://127.0.0.1:8001/api/`
- 管理后台：`http://127.0.0.1:8001/admin/`

### 3.3 环境变量模板（后端）

在 `backend/.env` 中配置（根据你的实际环境填写）：

```env
# Django
SECRET_KEY=please-change-me
DEBUG=True
ALLOWED_HOSTS=localhost,127.0.0.1

# MySQL
DB_NAME=iot_monitor
DB_USER=root
DB_PASSWORD=your_password
DB_HOST=127.0.0.1
DB_PORT=3306

# Redis / Channels / Cache
REDIS_HOST=127.0.0.1
REDIS_PORT=6379

# Celery
CELERY_BROKER_URL=redis://127.0.0.1:6379/0
CELERY_RESULT_BACKEND=redis://127.0.0.1:6379/0

# 机器人模块（可选，见 开发文档.md）
ERROR_RATE_CSV_PATH=/path/to/error-rate-csv-folder
```

## 4. 常用运维/数据导入

- 周结果导入命令说明见：`docs/weekly_result_integration.md`
- 定时导入脚本：`backend/scripts/setup_cron.sh`（会写入系统 crontab；建议先在测试机验证并评审脚本参数）

## 5. 常见问题（排查顺序）

- 前端接口 404/500：确认后端端口与 `frontend/vite.config.js` 代理目标一致（默认 `8001`）。
- WebSocket 不通：确认 Redis 可用，且 `REDIS_HOST/REDIS_PORT` 配置正确（Channels 使用 `channels_redis`）。
- Celery 不工作：确认 `CELERY_BROKER_URL` 指向可用 Redis；worker 是否启动。
- 数据库连接失败：确认 MySQL 账号密码、主机端口、库名正确。

## 6. 已知一致性提示（启动前建议确认）

- `README.md`/部分文档标注 Django 版本为 `6.0.1`，但 `backend/requirements.txt` 约束为 `Django>=3.2,<4.0`；请以你的实际依赖安装结果为准，必要时统一版本声明与依赖约束后再部署。
