# 接口文档（sg5.7）

## 1. 约定

- **Base URL**：`/api`
- **数据格式**：除下载/页面类接口外，均为 `application/json`
- **鉴权**：前端使用 `Token`（见 `frontend/src/utils/request.js`），请求头：
  - `Authorization: Token <token>`
- **分页**：
  - DRF 默认分页（如 `devices`、`robots/components`、`robots/high-risk-histories`）：`page`、`page_size`
- **时间格式**：
  - DRF `DateTimeField`：ISO8601（如 `2026-03-16T10:00:00+08:00`）
  - 例外：`/api/monitoring/export/` 使用 `YYYY-MM-DD HH:MM:SS`

## 2. 认证（/api/auth）

### 2.1 登录

- `POST /api/auth/login/`
- 请求体：
  - `username` string 必填
  - `password` string 必填
- 响应：
  - `message` string
  - `user_id` int
  - `username` string
  - `token` string

### 2.2 登出

- `POST /api/auth/logout/`
- 鉴权：需要 `Authorization: Token <token>`
- 响应：
  - 成功：`{ "message": "登出成功" }`
  - 失败：`{ "error": "登出失败" }`

### 2.3 当前用户信息

- `GET /api/auth/profile/`
- 鉴权：需要 `Authorization: Token <token>`
- 响应：
  - `user_id` int
  - `username` string
  - `email` string
  - `date_joined` datetime

## 3. 设备管理（/api/devices）

### 3.1 设备列表（分页）

- `GET /api/devices/devices/`
- 鉴权：需要
- 查询参数：`page`、`page_size`（可选）
- 响应：DRF 分页结构（`count/next/previous/results`），`results` 为 `DeviceList`

### 3.2 创建设备

- `POST /api/devices/devices/`
- 鉴权：需要
- 请求体：`DeviceCreate`（可带 `config`）

### 3.3 设备详情 / 更新 / 删除

- `GET /api/devices/devices/{id}/`
- `PUT/PATCH /api/devices/devices/{id}/`
- `DELETE /api/devices/devices/{id}/`
- 鉴权：需要

### 3.4 更新设备配置

- `POST /api/devices/devices/{id}/update_config/`
- 鉴权：需要
- 请求体：`DeviceConfig`（支持部分字段）
- 响应：更新后的 `DeviceConfig`

### 3.5 获取设备最新数据

- `GET /api/devices/devices/{id}/latest_data/`
- 鉴权：需要
- 响应：最新一条环境数据（无数据时 `404` + `{ "message": "暂无数据" }`）

### 3.6 切换设备状态

- `POST /api/devices/devices/{id}/toggle-status/`
- 鉴权：需要
- 请求体：
  - `status` string 必填：`online | offline | maintenance`
- 响应：`{ "message": "设备状态已更新为..." }`

## 4. 数据监测（/api/monitoring）

### 4.1 传感器数据列表

- `GET /api/monitoring/data/`
- 鉴权：需要
- 查询参数：
  - `device_id` int（可选，过滤设备）
  - `limit` int（可选，默认 `100`）
- 响应：列表（或在部分 DRF 配置下为分页结构）

### 4.2 传感器数据详情

- `GET /api/monitoring/data/{id}/`
- 鉴权：需要

### 4.3 获取所有设备最新数据

- `GET /api/monitoring/data/latest/`
- 鉴权：需要
- 响应：数组（每个元素包含 device 信息 + 最新传感器值）

### 4.4 设备上报数据

- `POST /api/monitoring/upload/`
- 鉴权：无（当前代码未强制），但要求设备必须为 `online` 才能上报
- 请求体：`SensorDataCreate`
- 响应：`{ "message": "数据上报成功", "data_id": int }`
- 备注：成功后会触发 Celery 任务 `alerts.tasks.check_alert_rules`

### 4.5 查询历史数据（分页）

- `GET /api/monitoring/query/`
- 鉴权：需要
- 查询参数：
  - `device_id` int 必填（设备表主键）
  - `start_time` datetime 必填（ISO8601）
  - `end_time` datetime 必填（ISO8601）
  - `data_type` string 可选（当前未用于过滤）
  - `page` int 可选，默认 `1`
  - `page_size` int 可选，默认 `50`
- 响应：`{ total, page, page_size, data: SensorData[] }`

### 4.6 导出数据（文件下载）

- `GET /api/monitoring/export/`
- 鉴权：需要
- 查询参数：
  - `device_id` int 必填（设备表主键）
  - `start_time` string 必填：`YYYY-MM-DD HH:MM:SS`
  - `end_time` string 必填：`YYYY-MM-DD HH:MM:SS`
  - `type` string 可选：`excel | csv`（默认 `excel`）
- 响应：文件流（Excel 或 CSV）

### 4.7 获取指定设备最近 100 条数据

- `GET /api/monitoring/realtime/{device_id}/`
- 鉴权：需要
- 路径参数：`device_id` 为设备表主键（int）
- 响应：`{ device: {...}, data: SensorData[] }`

### 4.8 统计数据

- `GET /api/monitoring/statistics/{device_id}/`
- 鉴权：需要
- 查询参数：
  - `range` string 可选：`24h | 7d | 30d`（默认 `24h`）
- 响应：
  - `summary`：聚合统计（avg/max/min/count）
  - `hourly`：按小时聚合数组（`hour/avg_temp/avg_humidity/avg_pm25`）

### 4.9 WebSocket（实时推送）

- `ws://<host>:<port>/ws/realtime/{device_id}/`
- 连接后可发送：`{ "type": "subscribe" }`，服务端回：`subscription_confirmed`
- 备注：当前仓库未实现生产者向组内推送 `sensor_update`（仅定义了 consumer）

## 5. 报警管理（/api/alerts）

### 5.1 报警规则（CRUD）

- `GET /api/alerts/rules/`
- `POST /api/alerts/rules/`
- `GET /api/alerts/rules/{id}/`
- `PUT/PATCH /api/alerts/rules/{id}/`
- `DELETE /api/alerts/rules/{id}/`
- 鉴权：需要

### 5.2 启用/禁用规则

- `POST /api/alerts/rules/{id}/toggle/`
- 鉴权：需要
- 响应：`{ "enabled": boolean }`

### 5.3 报警记录（只读）

- `GET /api/alerts/records/`
- `GET /api/alerts/records/{id}/`
- 鉴权：需要
- 查询参数（列表）：
  - `status` string 可选
  - `severity` string 可选
  - `device_id` int 可选

### 5.4 待处理报警

- `GET /api/alerts/records/pending/`
- 鉴权：需要

### 5.5 报警统计（聚合）

- `GET /api/alerts/records/stats/`
- 鉴权：需要
- 响应：`{ summary: {...}, by_severity: {...} }`

### 5.6 确认/解决报警

- `POST /api/alerts/records/{id}/acknowledge/`
- `POST /api/alerts/records/{id}/resolve/`
- 鉴权：需要
- 请求体：`{ "notes": string? }`

### 5.7 报警统计大盘

- `GET /api/alerts/statistics/`
- 鉴权：需要
- 查询参数：`range=24h|7d|30d`（默认 `7d`）

## 6. 机器人模块（/api/robots）

### 6.1 机器人组列表

- `GET /api/robots/groups/`
- 查询参数（可选）：
  - `start_date`：`YYYY-MM-DD`
  - `end_date`：`YYYY-MM-DD`
- 响应：`RobotGroup[]`（包含 `stats`）

### 6.2 机器人组件列表 / 详情 / 编辑

- `GET /api/robots/components/`（分页）
- `GET /api/robots/components/{id}/`
- `PUT/PATCH /api/robots/components/{id}/`
- 主要筛选参数（列表）：
  - `group`：车间 key
  - `tab`：`all | highRisk | history`（`history` 时请改查 `/api/robots/high-risk-histories/`）
  - `keyword`：关键字（robot/shop/reference/type/tech/remark）
  - `level`：支持逗号分隔
  - `markMode`：`zero | nonzero`
  - `axisKey` / `axisKeys`：`A1..A7`
  - `axisOk`：`1/0/true/false/ok/high/...`（详见代码解析）
  - `sort_by` / `sort_order=asc|desc`
- 编辑请求体（常用）：
  - `reference` string（写入时使用该字段；输出用 `referenceNo`）
  - `number` number
  - `mark` int
  - `remark` string
  - `level` string（`H|M|L|T|C`；传空字符串会被忽略）
- 备注：更新后会尝试同步本地 `机器人配置文件.csv`

### 6.3 BI 机器人选择列表（含 SG 数据库回退）

- `GET /api/robots/components/bi_robots/`
- 查询参数：`group`、`keyword`
- 响应：`{ results: [{ value, label, robot_id, shop, group_key }] }`

### 6.4 BI 数据时间范围

- `GET /api/robots/components/time_range/?robot=<table>`

### 6.5 关节错误率趋势图（base64）

- `GET /api/robots/components/{id}/error_trend_chart/?axis=1`
- 响应：`{ success, chart_data, axis, robot }`

### 6.6 导入 weeklyresult.csv（同步/异步）

- `POST /api/robots/components/import_csv/`
- `POST /api/robots/components/import_csv_async/`
- `GET /api/robots/components/import_csv_status/?task_id=...`
- 请求体（导入）：
  - `folder_path` string 可选
  - `project` string 可选（默认 `reuse`）
  - `file_path` string 可选
  - `use_mysql_load_data` boolean 可选

### 6.7 机器人统计摘要

- `GET /api/robots/components/stats_summary/`

### 6.8 风险事件（RiskEvent）

- `GET /api/robots/risk-events/`
- `GET /api/robots/risk-events/{id}/`
- `POST /api/robots/risk-events/{id}/acknowledge/`
- `POST /api/robots/risk-events/{id}/resolve/`
- `GET /api/robots/risk-events/statistics/`
- 查询参数（列表）：`group`、`status`、`severity`

### 6.9 关键轨迹检查（Gripper Check）

- `POST /api/robots/gripper-check/execute/`
- `GET /api/robots/gripper-check/status/`
- `GET /api/robots/gripper-check/latest/`
- `GET /api/robots/gripper-check/robot_tables/?group=&keyword=`
- `GET /api/robots/gripper-check/config_template/`

### 6.10 历史高风险列表（快照）

- `GET /api/robots/high-risk-histories/`（分页）
- 查询参数：`group`、`level`、`keyword`、`markMode`、`axisKey/axisKeys/axisOk`、`sort_by/sort_order`

### 6.11 Reference Dict（reference→number）

- `GET /api/robots/reference-dict/?robot=...`
- `POST /api/robots/reference-dict/refresh/`（异步任务）
- `GET /api/robots/reference-dict/refresh-status/?task_id=...`
- `GET /api/robots/reference-dict/resolve/?robot=...&reference=...`

### 6.12 BI 页面与图表文件

- `GET /api/robots/bi/?robot=<table>&embed=0|1&program=&axis=&start_date=&end_date=`
  - 返回 HTML（用于 iframe 或直接打开）
- `GET /api/robots/charts/<filename>`
  - 返回 PNG 文件（从配置的图表输出目录读取）

### 6.13 数据刷新与日志

- `GET /api/robots/last_sync_time/`
- `GET /api/robots/refresh_logs/?limit=12`
- `GET /api/robots/bi_logs/?limit=8`

### 6.14 关键路径告警（外部 keypath 数据库）

- `GET /api/robots/keypath-warnings/?robot=&limit=10`

### 6.15 组件编辑“二次认证”（独立于主登录）

- `POST /api/robots/auth/verify/`
- `GET /api/robots/auth/status/?version=...`
- `POST /api/robots/auth/increment-version/`
- `GET /api/robots/auth/users/`
- `POST /api/robots/auth/change-password/`
- `POST /api/robots/auth/create-user/`
